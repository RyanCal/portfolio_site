Dynamic Lead Time Calculator

SQL Data Analysis E-commerce Process Automation

Developed a sophisticated SQL-based system to automatically calculate and update product shipping estimates across an entire e-commerce platform. This solution:

  • Processes historical shipping data to generate accurate delivery estimates
  • Handles multiple vendor types (OEM, Wholesale Distributors, 3PL)
  • Incorporates business rules for special cases and made-to-order items
  • Improves customer experience by providing accurate shipping expectations

Key Implementation Snippet

WITH BaseOrderData AS (
    SELECT DISTINCT
        o.ordr_id,
        oi.variant_id,
        p.vendor_id AS OEM_Number,
        oi.vendor_id,
        oi.fulfillment_vendor_id,
        -- Business days calculation
        DATEDIFF(DAY, po.purchase_ordr_date, os.create_date)
        - (DATEDIFF(WK, po.purchase_ordr_date, os.create_date) * 2)
        - CASE WHEN DATEPART(dw, po.purchase_ordr_date) = 1 THEN 1 ELSE 0 END
        + CASE WHEN DATEPART(dw, os.create_date) = 1 THEN 1 ELSE 0 END AS D2SH
    FROM [P4].[dbo].[ordr] o
    JOIN [P4].[dbo].[ordr_item] oi ON o.ordr_id = oi.ordr_id
    -- Additional joins omitted for brevity
    WHERE o.sale_date > DATEADD(DAY, -@SalesHistoryDays, GETDATE())
),

ShippingMetrics AS (
    SELECT
        variant_id,
        COUNT(*) AS ordrs_per_variant,
        SUM(CASE WHEN vendor_id = fulfillment_vendor_id THEN 1 ELSE 0 END) AS OrdrOEM,
        AVG(CASE WHEN vendor_id = fulfillment_vendor_id THEN D2SH END) AS Avg_D2SH_OEM,
        AVG(CASE WHEN vendor_id != fulfillment_vendor_id THEN D2SH END) AS Avg_D2SH_WD
    FROM BaseOrderData
    GROUP BY variant_id
)

Fulfillment Priority Optimization System

SQL Supply Chain Automation Process Optimization

Built an automated SQL system to optimize vendor selection across multiple fulfillment channels, factoring in costs, margins, and business rules. This solution:

  • Processes over 100,000 SKUs daily to determine optimal fulfillment paths
  • Incorporates real-time inventory data from multiple suppliers
  • Handles complex business logic including vendor-specific rules and promotions
  • Optimizes for both cost and delivery speed

Core Logic Implementation

WITH BaseOrderData AS (
    SELECT DISTINCT
        variant_id,
        fulfillment_vendor_id,
        CASE
            WHEN unit_cost = 99999.99 THEN
                (ROW_NUMBER() OVER (PARTITION BY variant_id
                    ORDER BY unit_cost DESC, fulfillment_vendor_id)) + 79
            ELSE
                ROW_NUMBER() OVER (PARTITION BY variant_id
                    ORDER BY unit_cost, fulfillment_vendor_id)
        END AS new_sortpriority,
        CASE
            WHEN sortpriority <> CASE
                WHEN unit_cost = 99999.99 THEN
                    (ROW_NUMBER() OVER (PARTITION BY variant_id
                        ORDER BY unit_cost DESC, fulfillment_vendor_id)) + 79
                ELSE
                    ROW_NUMBER() OVER (PARTITION BY variant_id
                        ORDER BY unit_cost, fulfillment_vendor_id)
            END THEN 1
            ELSE 0
        END AS change
    FROM vendor_fulfillment_data
    WHERE display_status_id NOT IN ('del','zaa','z')
),

/* Key business logic handling vendor priorities */
SELECT
    variant_id,
    fulfillment_vendor_id,
    new_sortpriority AS sortpriority,
    change
FROM BaseOrderData
WHERE change > 0
    AND CASE
        WHEN vendor_type = 'OEM' AND margin >= threshold THEN 1
        WHEN inventory_available > 0 AND lead_time <= max_lead_time THEN 1
        ELSE 0
    END = 1
ORDER BY variant_id, sortpriority
